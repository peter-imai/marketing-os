Switch to **alignment mode** — interview the operator until shared understanding is reached, with a recommended answer for every question.

This command layers on top of any operating-mode command. It does NOT load client or system context — assume that's already loaded. Use to re-engage alignment mid-session if drift is suspected, or as an explicit handle when the default Drafting rule's "Align first" gate didn't fire.

## Invocation phrases

Map all of the following to this protocol — don't improvise a different interrogation style:
- `/align`, `/grill-me`, `/grill`
- "grill me", "grill me on this"
- "ask me questions", "interview me", "interrogate me"
- "pressure-test my thinking", "challenge me on this"
- "align with me", "let's align", "get aligned first"

If the operator uses any of these phrases — even without the slash — run this protocol.

## The Protocol

You are running the alignment protocol.

Interview the operator about every load-bearing decision in this work until shared understanding is reached. Walk down each branch of the decision tree, resolving dependencies between decisions one by one.

**For each question, provide your recommended answer.** Don't ask open questions without a proposed default — the proposal is the load-bearing mechanic. The operator confirms or redirects faster than they can answer open questions cold.

**Continue until the operator explicitly authorizes execution** ("proceed", "build it", "go", "draft it", "ship", or equivalent). Acknowledgment is not authorization — "ok" or "that's fine" does not close the alignment loop unless followed by explicit go-ahead.

**Sub-decision re-fire:** If you encounter a sub-decision mid-execution that changes the scope or approach the operator aligned on, re-engage this protocol for that branch. Tactical choices inside agreed scope don't re-engage.
