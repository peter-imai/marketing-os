# Step 3: Compose

**Goal:** Write the piece using the two-pass method. Structure lock first, then draft. Operator reviews in batches.

---

## Rules

- **Two-pass is the default.** Structure first, then draft. Don't skip it — it fixes opening architecture and prevents template patterns.
- **Compose from the operator's language.** Source material docs are the primary linguistic source. The operator's phrases, metaphors, and framings should appear in the output. Sharp lines from messaging docs nearly verbatim when they fit. When you need language that isn't in the docs, flag it: "I need language for [concept] — what would you say?"
- **Work in sections, not all at once.** Present 2-3 sections at a time. Get feedback. Iterate before moving forward.
- **Respect the format playbook.** The playbook specifies structure, length, and conventions for this format. Follow it.

---

## Instructions

### Pass 1: Structure Lock

Before writing any body content:

1. **Write the opening line.** Follow the format playbook's opening formula. For data writeups: action + result + assessment. For email: consequence-first (the most important thing in the first substantive sentence). For persuasive: the hook that earns the next sentence.

2. **List section headers** in reader-priority order. For data writeups: results before process. For email: substance before logistics. For persuasive: follows the Key Moves from the Copy Intent.

3. **Set the length budget.** State it explicitly: "Targeting [N] words / [N] paragraphs / [N] pages." The playbook has guidance. State it so you can check against it later.

4. **Verify:** Does the opening lead with what matters? Are results/substance before methodology/context? Does the structure follow the argument's logic, not a formula?

**Present the structure lock to the operator.** "Here's the opening and structure I'm proposing. Look right?" Quick confirmation — not a long discussion unless the structure is wrong.

### Pass 2: Draft

5. **Fill in each section.** Work through the locked structure section by section.

6. **For each section, self-check:**
   - Does this sound like the operator with convictions, or like a marketing template?
   - Can I trace every significant claim to the loaded context?
   - Does the structure advance the argument, or is it decorative?
   - Would the reader's emotional state match the arc at this point?
   - Does this section earn its place, or could it be cut?
   - Am I within the length budget?

7. **Present 2-3 sections at a time.** Get operator feedback. Iterate before moving forward. This prevents bad structure from embedding across a full piece.

8. **After all sections drafted and approved:** Present the full piece as one continuous document. Does the arc hold end-to-end? Does the ending earn the CTA (if applicable)?

### For Email (Streamlined)

Email is typically short enough to compose in one pass after the structure lock:
- Lock the opening + section headers (Pass 1)
- Draft the full email (Pass 2 — single batch, not section by section)
- Present the complete draft with metadata header

```
---
To: [names + emails]
CC: [if applicable]
From: [send account]
Subject: [draft subject — operator can override]
Reply-To: [thread info if replying]
---

[Full email body]
```

---

## Anti-Patterns

- **Do NOT start with structure.** "Here are 8 sections" is backwards. Argument first (Step 2), structure second (Pass 1), content third (Pass 2).
- **Do NOT fill in templates.** Sales page formulas (PAS, AIDA, StoryBrand) are reference tools, not fill-in-the-blank. The operator's convictions and the reader's psychology determine the arc.
- **Do NOT generate claims without context grounding.** If a claim isn't in the loaded docs and the operator hasn't said it, don't make it up. Ask.
- **Do NOT present full long-form copy in one dump.** Work in sections. Get feedback. Iterate. (Email is the exception — short enough for one pass.)
- **Do NOT continue writing if the operator says the voice is wrong.** Stop. Re-read the voice kernel and source material. Restart the section.
- **Do NOT let previous copy influence new copy.** If the operator says "start fresh," the intent is the ONLY input to composition. No echoes of old drafts.

---

## Success Criteria

- [ ] Structure locked (opening + headers) before drafting
- [ ] Length budget set and respected
- [ ] Operator reviewed sections in batches (or full email in one pass)
- [ ] Full piece presented as one continuous document
- [ ] Operator approved the draft (may request revisions)
